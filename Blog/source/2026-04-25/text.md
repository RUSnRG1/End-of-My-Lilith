# レイアウトにこだわりたいの！
　小説を書く上でLaTeXにはこんなメリットが……というのは[このサイト](https://tadeku.net/94393/)に大体書いてあるので読んでください。ここではもっと、同人ならではの利点について語りたい。
　同人小説を読んでいていつも感じるのが、漫画と比べて本文のレイアウトにこだわる人が少ないなと。例えば装丁のデザインやカバーを凝ったり、頒布方法を工夫したりする方もいらっしゃるんですが、本文自体はごく普通に文字が並んでいるだけ、みたいな。やりたくないですか？　いきなり文字回転させるとか。フォント変えるとか。急にサイズ変えたりボールドにするとか。ルビで遊ぶとか。
　ただ、思いついても実現するのが難しい、というのはあると思います。漫画は白紙に自由に描きたいもの描けますが、小説だと決められた場所に字を置く以外のことが難しいと。でもね、あなたが想像力を働かせて生み出したその表現、実装できます。そう、LaTeXならね。
　というわけで、以下は私が実際にやった例です。

## 1. 場面転換で画像使う
　場面転換する際どうしますか？　空白行を入れるとか、「＊」を入れるなどですますことが多いと思います。
　先のサイトの小技にもありますが、「＊」の代わりに画像使うという案。簡単に実装できます。
　私の二冊目の小説「左目の追憶」では、一人称を蓮子とメリーとで行き来させていました。そこで、「いまこの文章の一人称はどちらなのか」の転換を、一目で表現できるようにしました。

<figure style="text-align:center;">
  <img src="Blog/source/2026-04-25/1.png"  width="400"/>
  <figcaption style="font-size:small;">「左目の追憶」より</figcaption>
</figure>

　今、誰の一人称なのか一目瞭然ですね。またこの小説は七夕坂夢幻能関連の話しですが、メリーの帽子の紐が……みたいな遊びかたもできるわけです。
　コードとしては、先のサイトを参考にしつつも複数画像を扱えるように改造しました。
```
\newcommand{\imagebreak}[1]{
  \hspace*{2\zw}\mbox{\yoko\includegraphics[width=2\zw]{#1}}\par
 \mbox{}\par
}
```
として、本文の中に
```
\imagebreak{merrybreak.png}
　蓮子が駆け足で部屋を出ていくと……
```
のように画像名を入れる。これでその画像を場面転換として使うことができます。素敵！

　え、別にWordでも都度画像を挿入すればいいだけ？そうですね。

## 2.トビラ
　章とか部の始まりとなる区切りのページをトビラと言いますね。このトビラ、どうやって作りますか？
　二次創作なら絵をかいて挿入するのが一番いいと思うんですが、本文推敲の方に時間がかかると描いてられなくて、字だけになりがち。LaTeXならこのトビラも、多少は凝れます。

「左目の追憶」の時は
```
format={\null\vspace{17em}\hspace{6em}{\Huge 第\thepart 章}} 
```
とすることで

  <img src="Blog/source/2026-04-25/2.png"  width="400"/>
  <figcaption style="font-size:small;">「左目の追憶」のトビラ</figcaption>
</figure>
シンプルイズベスト。


「女帝」の時は、各章に章タイトルをつけていました。
```
format={\null\vspace{14em}\hspace{3em}{\Large 第\thepart 章　　　\Huge ##1}}
```

  <img src="Blog/source/2026-04-25/3.png"  width="400"/>

こんな感じ。
　そもそもまだトビラがある本をそんな作ってないのですが、秋の本は結構凝る予定です。

　ちなみにある商業本で、各章ごとに別々の（章に関係ある小物の）イラストと、その下に短い詩を入れる、といったトビラデザインがありました。もちろん実装できますよ。骨組みだけは作ったので、いつかやってみたいなーと思ったり。

　え、Wordでもトビラページを都度作ればいいだけ？　そうですね。

## 4.註釈
　註釈を入れられるようになると、本文だけでは絶対伝わらない内容も説明できます。例えば自分の好きな曲をいきなり作品内で流して、作者としては「こんないい曲なんですよ」と主張しながら、キャラには「なんだこの曲」って苦い顔をさせられるわけです。
　私は註釈芸人なので乱用しています。小説「女帝」では註釈が実は作者ではなく作品内キャラによるものだということにしているし、「(∀λ∈Λ,A_λ≠∅)⇒∏_{λ∈Λ}A_λ≠∅」に至っては註釈を本編にしています。何でもできるんですね。
　さて、LaTeXはもともと論文向けのものなので、註釈はもはやパッケージとして用意されてます。
　例えば
```
「おお、ここでFour Brothers\endnote{ジミー・ジュフリー『Four Brothers』。歌詞はThe Manhattan Transferによるもの}を演奏するのか。～
```
という風に、本文中に\endnote{註釈}と入れます。そして註釈をまとめて置きたい場所で
```
\theendnotes%
```
のコードを入れれば
<figure style="text-align:center;">
  <img src="Blog/source/2026-04-25/67.png"  width="300"/>
  <figcaption style="font-size:small;">左：＊4と註釈が自動採番される。右：註釈内容が後でまとめて表示される</figcaption>
</figure>

こうなるってわけ。\theendnotesを章末におけば章末に註釈がでるし、文末に置けば文末に出る。
　自動採番されて便利ですね。章末にカウンターをリセットすれば、次の章からはまた1からカウントされるようになる。便利ですね。

　え、Wordでも註釈は入れられる？　そうですね。

## 5.横書き併用
　縦書きの本文内に、いきなり横書き入れたくないですか？　メリーって外国人ですよ？　彼女にいきなり英語を喋らせて、突然横書き小説を始めたくなりますよね！（やろうとした小説は凍結した）

\end{minipage}

# 結局のところ
　ここまでLaTeXの素晴らしき世界をご紹介しました。で、お勧めかと言われると、全くお勧めしないです（じゃあなんだこのブログのタイトルはいったい）。

　LaTeXはね、プログラミングなんですよ。ええ、プログラミング。ほら、この言葉を聞いた瞬間に全身に発疹が出てひっくり返ってしまう方で溢れていることでしょう。
　それは言いすぎとしても、コーディングを楽しいと思わない方も沢山いると思います。ならばデバッグに時間かけるより、小説の内容吟味に時間を使った方が有意義だと思います。
　また、レイアウトを沢山いじれるのがメリットとしましたが、多分InDesignや一太郎でもなんとかなると思います。知らんけど。そっちの方がやってる人も沢山いると思うので、聞ける人が多くていいんじゃないですかね。
　ただやっぱり全部を自分で書くと、俺が作ったんだよなーって愛着が湧きます。楽しいですよ、LaTeXで遊ぶの。イドラデウスの気分を味わいたい方は、是非やってみてはいかがでしょうか？

　それでデバッグで時間取られて新刊落としても、私は一切の責任を取りませんけどね。
