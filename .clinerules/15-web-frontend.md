# Web frontend rules

## 前提
- 本HPはGithub pagesを使って公開している
- Live Serverをつかってテストしているので、IPはそれ用と通常公開用の二つがある
- nengazyoのフォルダは全く使われてないですが、特別なフォルダなので残すこと

## General policy
- まず全体を監査し、修正候補を整理してから着手する。
- 既存デザインや既存構造を尊重し、不要な全面改修は行わない。
- 変更は小さく保ち、影響範囲の大きい修正は慎重に扱う。
- HTML/CSS/JS の責務を分け、場当たり的な修正を避ける。
- レスポンシブ、アクセシビリティ、コンソールエラーを確認する。
- 既存の部品や記述を再利用できる場合はそれを優先する。

## Project-specific additions
- この節は、最初の全体監査のあとに追記する。
- ディレクトリ構成
- 命名規則
- 共通CSSや共通JSの扱い
- 触ってよいファイル / 慎重に触るファイル
- レスポンシブの主要ブレークポイント
- 既知の崩れやすい箇所
