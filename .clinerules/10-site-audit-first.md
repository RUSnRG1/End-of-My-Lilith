# Site audit first

## First step of every new site task
- 新しいサイト案件を開始したら、いきなり編集せず、まず全体を把握する。
- 最初の返答では、ディレクトリ構造、主要HTML、共通CSS、共通JS、画像や外部依存の有無を確認する。
- その後、改善候補を「緊急性が高いもの」「品質改善」「将来的な改善」に分けて整理する。

## Audit output format
- 監査結果は次の順序で出す。
  1. サイト構成の把握
  2. 問題点または改善候補
  3. 先に直すべき候補
  4. 今すぐ触らない方がよい箇所
- 問題点は、見た目、保守性、アクセシビリティ、レスポンシブ、重複、JSの責務の観点で挙げる。

## Before implementation
- 監査結果を出した後で、ユーザーの指示に応じて次のどちらかに進む。
  - 監査結果に沿って修正する
  - 監査結果とは別に、新機能や別修正を行う
- 監査結果は提案であり、ユーザーの明示指示が優先である。
